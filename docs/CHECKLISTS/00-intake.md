# Checklist: Intake (before spawning specialists)

## Inputs
- [ ] Project selected and path confirmed
- [ ] `PROJECT.md` exists and is up to date
- [ ] `BRAND.md` exists (or explicitly N/A)

## Task framing
- [ ] Clear problem statement
- [ ] Success metric / Definition of Done
- [ ] Constraints (time/budget/tech/compliance)
- [ ] Known dependencies & integrations

## Output contract
- [ ] Exact output folder specified (where files must be written)
- [ ] Exact filenames requested (e.g., `docs/specs/012-foo.md`)
- [ ] Handoff target specified (which agent is next)

## Risk flags
- [ ] Any legal/regulatory claims flagged for verification
- [ ] Any secrets/credentials handling requirements stated
