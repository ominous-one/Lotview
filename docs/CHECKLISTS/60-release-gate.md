# Gate: Release (post-merge / pre-deploy)

Pre-release:
- [ ] Versioning decided (tag/release notes)
- [ ] Migration plan executed/verified (if any)
- [ ] Rollback plan documented

Deploy:
- [ ] Deploy steps followed from runbook
- [ ] Smoke tests pass

Post-release:
- [ ] Monitor key metrics/logs
- [ ] Incident channel/owner set for first 24h
- [ ] `docs/STATUS.md` updated with outcome
