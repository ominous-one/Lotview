# Gate: Executioner (QA approval)

Test execution:
- [ ] Happy path verified
- [ ] Failure paths verified (timeouts, bad input, missing auth)
- [ ] Edge cases verified
- [ ] Concurrency/retry/idempotency tested when relevant

Reporting:
- [ ] Repro steps written for every failure
- [ ] Logs/stack traces captured where useful
- [ ] Severity assigned to issues
- [ ] `docs/STATUS.md` updated

Handoff:
- [ ] `docs/HANDOFFS/executioner-to-engineer.md` if defects found
- [ ] Otherwise ready for `reviewer` gate
